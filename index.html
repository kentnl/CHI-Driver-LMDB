Read Performance is slightly better with Single Transactions, but both Single and Mutliple Transaction modes are comparable to FastMmap.
MultiTransactions + Sync is slow
MultiTransactions + NoMetaSync is slightly faster
Single Transactions + Sync is 3rd Slowest
Single Transactions + NoMetaSync is slightly faster
MultiTransactions + NoSync is 5th Slowest
MultiTransactions + NoAny*Sync is not much better ( Any = NoMetaSys + NoSync )
SingleTransactions + NoSync is fastest
SingleTransactions + NoAny*Sync is about as good ( Any = NoMetaSys + NoSync )
Somehow, both end up slightly faster than FastMmap
